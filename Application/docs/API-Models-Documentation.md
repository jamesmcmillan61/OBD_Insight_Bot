# API Models Documentation

## 1) Chat API Response

The `ChatApiResponse` model represents the JSON response returned from the chatbot API service. It is used to deserialize the API response into a strongly-typed C# object so the application can access chatbot output and metadata reliably.

### Purpose
- Stores the chatbot’s reply message and supporting metadata (session info, timing, detected intent).
- Uses `System.Text.Json` attributes to map JSON field names to C# properties.

### Properties

- **Response** (`string`)
  - The main text response generated by the chatbot.
  - Maps from JSON field: `"response"`.

- **SessionId** (`string`)
  - The session identifier returned by the chatbot API, used to link requests and responses.
  - Maps from JSON field: `"session_id"`.

- **Timestamp** (`string`)
  - Timestamp of when the response was generated by the API.
  - Maps from JSON field: `"timestamp"`.

- **IntentDetected** (`string?`)
  - Optional field containing the detected intent/classification for the user message.
  - Maps from JSON field: `"intent_detected"`.

- **ProcessingTimeMs** (`double`)
  - The time taken by the chatbot service to process the request (in milliseconds).
  - Maps from JSON field: `"processing_time_ms"`.

## 2) Error View Model

The `ErrorViewModel` is a simple model used to display error information in the application's error view. It helps provide a request identifier that can be used for debugging and tracing issues.

### Purpose
- Stores a request ID associated with an error page.
- Provides a helper property to determine whether the request ID should be shown in the UI.

### Properties

- **RequestId** (`string?`)
  - The unique identifier for the current request.
  - This may be used to trace logs and diagnose errors.

- **ShowRequestId** (`bool`)
  - Returns `true` if `RequestId` is not null or empty.
  - Used by the error view to conditionally display the request ID to the user.

## 3) User Chat Item

The `UserChatItem` model represents a single chat message stored in the database. Each chat item belongs to a specific `UserSession` and is used to persist the conversation history between the user and the chatbot.

### Purpose
- Stores individual chat messages (user, bot, or system).
- Links each message to a `UserSession` using a foreign key.
- Maintains message ordering using `ChatPosition` so the chat can be displayed in sequence.

### Properties

- **Id** (`Guid`)
  - Primary key for the chat item.
  - Generated as a new GUID when the object is created.
  - Marked with `[DatabaseGenerated(DatabaseGeneratedOption.None)]` meaning the application generates the value (not the database).

- **UserSessionId** (`Guid`)
  - Foreign key that links this chat message to a specific `UserSession`.
  - Marked as `[Required]`.

- **UserSession** (`UserSession`)
  - Navigation property for Entity Framework.
  - Allows access to the parent session entity.

- **ItemBy** (`ChatSender`)
  - Indicates who sent the message (`User`, `Bot`, or `System`).
  - Marked as `[Required]`.

- **ChatPosition** (`int`)
  - The message index within the session conversation.
  - Used to order messages correctly when displayed.
  - Validated using `[Range(0, int.MaxValue)]`.

- **SentAtUtc** (`DateTime`)
  - Timestamp of when the message was created/stored.
  - Defaults to `DateTime.UtcNow`.
  - Marked as `[Required]`.

- **ChatContent** (`string`)
  - The text content of the chat message.
  - Marked as `[Required]`.
  - Limited to a maximum length of **4000 characters** using `[MaxLength(4000)]`.

### Constructors

- **Private parameterless constructor**
  - Required by Entity Framework to materialize objects from the database.

- **UserChatItem(Guid userSessionId, ChatSender itemBy, string chatContent, int chatPosition)**
  - Main constructor used when creating new chat messages.
  - Ensures `chatContent` is not null (throws `ArgumentNullException` if invalid).
  - Assigns session ID, sender type, message content, and chat position.

---

### ChatSender Enum

The `ChatSender` enum defines the possible sender types for a chat message:

- **User** (`1`)
- **Bot** (`2`)
- **System** (`3`)

## 4) User OBD Data

The `UserOBDData` model represents vehicle and sensor data uploaded by the user (via CSV). Each record is linked to a specific `UserSession` and stores both vehicle metadata and diagnostic/sensor readings.

### Purpose
- Stores the parsed contents of an uploaded OBD CSV file.
- Links uploaded vehicle data to the session that uploaded it.
- Provides vehicle context (e.g., make/model/year, RPM, speed, trouble codes) used to improve chatbot responses.

### Properties

- **Id** (`Guid`)
  - Primary key for the OBD data record.
  - Generated as a new GUID when the object is created.
  - Marked with `[DatabaseGenerated(DatabaseGeneratedOption.None)]`, meaning the application generates the ID.

- **UserSessionId** (`Guid`)
  - Foreign key linking this OBD data record to a `UserSession`.
  - Marked as `[Required]`.

- **UserSession** (`UserSession`)
  - Navigation property for Entity Framework.
  - Allows access to the parent session record.

- **UploadedAt** (`DateTime`)
  - Timestamp of when the OBD data was uploaded/created.

---

## 5) User Session

The `UserSession` model represents a single user interaction session within the application. It is used to group together uploaded OBD data and chat history so that each conversation is isolated and tied to a specific session.

### Purpose
- Creates a unique session identifier used across the application.
- Tracks when a session started and when it was last active.
- Stores the file path of the uploaded CSV file linked to the session.

### Properties

- **Id** (`Guid`)
  - Primary key for the session.
  - Marked with `[DatabaseGenerated(DatabaseGeneratedOption.Identity)]`.
  - Used as the main identifier to link:
    - chat messages (`UserChatItem`)
    - uploaded vehicle data (`UserOBDData`)

- **SessionStartTime** (`DateTime`)
  - Stores the time the session was created (start of user interaction).

- **lastUseTime** (`DateTime`)
  - Stores the last time the session was active.
  - Used for inactivity cleanup (e.g., removing sessions older than a set time limit).

- **UploadedDataPath** (`string?`)
  - Stores the server file path of the uploaded CSV file.
  - Nullable because a session may exist before a user uploads data.
